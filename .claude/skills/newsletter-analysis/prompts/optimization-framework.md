# Optimization Framework

## Analysis Dimensions

When generating recommendations, evaluate along these dimensions:

### 1. Subject Line Optimization (drives Opens)
- **Length:** Optimal word count based on data
- **Brand usage:** Which brands to lead with, which to rotate, which to avoid
- **Pattern:** Possessive, numbers, questions, negative framing — what works?
- **First word:** What entity/word should the subject start with?
- **Specificity:** Product names vs company names vs generic terms

### 2. Content Mix (drives CTR)
- **Topic balance:** What % of stories should be creative/art vs model releases vs business?
- **Actionable content:** Tutorial/prompt/tool links vs pure news reporting
- **Link types:** What types of links actually get clicked?
- **Section effectiveness:** Which newsletter sections drive the most engagement?

### 3. Structural Changes (drives both)
- **Section order:** Should actionable content come before or after news?
- **New sections:** "Try This", "Tool of the Day", etc.
- **Section removal:** What to cut if it underperforms?
- **Story count:** Is 3 stories optimal or should it change?

### 4. Send Strategy (drives Opens)
- **Day of week:** Best performing send days
- **Time of day:** If data available
- **Frequency:** Are there diminishing returns on daily sends?

### 5. Audience Health
- **Unsubscribe trends:** Is content driving people away?
- **Subscriber growth vs churn:** Net trajectory
- **Spam complaints:** Any alarming patterns?

## Recommendation Format

Each recommendation must include:
1. **What to change** — specific, actionable instruction
2. **Why** — the data point that supports it (with numbers)
3. **Expected impact** — estimated open/CTR change based on historical patterns
4. **Effort** — Low/Medium/High to implement
5. **How to measure** — what to track to validate the change

## Priority Rules

1. Changes that improve BOTH opens AND CTR take highest priority
2. Low-effort changes take priority over equal-impact high-effort changes
3. CTR improvements are more valuable than open improvements (monetization correlation)
4. Never recommend changes that sacrifice opens for CTR or vice versa by more than 1%
